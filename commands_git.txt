Чтобы инициализировать git в свой проект
git init



Чтобы посмотреть на версию гит
git --version



Перемещение по проекту:
чтобы зайти в какую-то директорию - cd test



Чтобы выйти из директории - cd ..



Чтобы показать содержимое нашей директории
ls



Чтобы в терминале открыть ФАЙЛ
nano commands_git.txt



Статус нашего git
git status



Чтобы добавить мои вещи в GIT-Репозитрий
git add название_файла_который_добавляем_в_гит - Добавляет в гит указанный файл
git add . - Добавляем все файлы в гит



Чтобы сохранить сделать снимок нашего проекта мы будем использовать команду
Т.е сохранить наш проект в GIT
git commit -m 'Описание того, что у нас случилось'



Чтобы увидеть историю изменений наших коммитов
git log - ALT + 9 | COMMAND 9



Покажет все наши изменения и файлы которые мы изменили в коммите
git show



Чтобы посмотреть изменений до коммита
git diff [name_file]



Мы можем вернуть наш файл в состояние последнего коммита
git restore [name_file]



Мы можем удалить наш из индекса
git rm [name_file]
git rm --cached [name_file]



Чтобы вообще удалить файл
rm -rf [name_file]



Чтобы вернуть наш проект в какой-то определенный коммит
git reset commit_hash



Откатывается и переводим все последующие коммиты в индекс
git reset --soft commit_hash



Откатывается и переводим все последующие коммиты в неотслеживаемую зону
git reset --mixed commit_hash



Самое опасное. После того как мы откатим коммит мы не сможем вернуть наши исходные файлы
git reset --hard commit_hash



Чтобы создать новую ветку
git branch [name]



Чтобы изменить название ветки
git branch -m [old_name] [new_name]



Чтобы посмотреть на все ветки нужно
git branch



Чтобы зайти в ветку
git checkout [branch_name]





Чтобы стянуть изменения с каких-то веток (сделать слияние веток)
git merge [branch_name]
